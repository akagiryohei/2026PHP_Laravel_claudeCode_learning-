# ドキュメント構成

> **どこに何があるか迷ったらここを見る。**

## ファイル一覧

```
Doc/
├── README.md                         # このファイル（全体インデックス）
├── context.md                        # プロジェクト経緯・目的・環境・進め方の方針
└── learning-plan/
    ├── overview.md                   # 全フェーズの進捗表（フェーズ間の俯瞰用）
    ├── phase1/
    │   └── checklist.md             # Phase1 の実装チェックリスト・ステータス
    ├── phase2/
    │   └── checklist.md             # Phase2 の実装チェックリスト・ステータス
    ├── phase3/
    │   ├── checklist.md             # Phase3 の実装チェックリスト・ステータス
    │   └── guide.md                 # Phase3 の学習解説（リレーション・マイグレーション・クエリビルダー）
    └── phase4/
        └── checklist.md             # Phase4 の実装チェックリスト・ステータス
```

## ファイルの役割

| ファイル | 役割 | 更新タイミング |
|---------|------|--------------|
| `context.md` | プロジェクトの背景・ユーザースキル・環境情報 | 方針変更・環境変更時 |
| `overview.md` | 全フェーズの進捗一覧 | フェーズ完了時 |
| `phaseN/checklist.md` | そのフェーズの実装項目と完了チェック | 実装を進めるたびに |
| `phaseN/guide.md` | そのフェーズで学ぶ概念・コード例の解説 | フェーズ開始時に作成 |

## 命名ルール

- **`checklist.md`** — 進捗管理用。「何を作るか」「終わったか」を管理する。
- **`guide.md`** — 学習解説用。「なぜそう書くのか」「どう動くのか」を説明する。
